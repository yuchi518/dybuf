# dybuf Java package

Java implementation of the [`dybuf`](../c/dybuf.h) dynamic buffer library. The C
implementation remains the canonical wire-format reference; Java must pass the shared
golden fixtures generated by `tools/generate_fixtures.sh`.

## Key features

- Pure Java `DyBuf` implementation under package `oyo.lol.util`.
- Supports reading and writing fixed-width integers, booleans, raw byte payloads,
  compact unsigned/signed varints, and fixed-width float/double payloads.
- Exposes typdex markers so higher-level protocols can tag fields without a
  protobuf-style schema.
- Provides JSON-equivalent helpers for binary JSON round-tripping with shared key
  dictionaries.
- Includes a plain `javac`/`java` golden fixture runner without Maven or Gradle.

## Quick start

```java
import oyo.lol.util.DyBuf;

DyBuf buf = new DyBuf(64);
buf.putShort((short) 0x1234)
   .putInt(0xDEADBEEF)
   .put(true);

buf.flip();

System.out.println(Integer.toHexString(buf.getShort() & 0xffff)); // 1234
System.out.println(Integer.toHexString(buf.getInt()));            // deadbeef
System.out.println(buf.getBoolean());                             // true
```

`position()`, `limit()`, `capacity()`, `flip()`, `rewind()`, and `compact()` follow
the same cursor-style model as the C library.

## Variable-length integers

`DyBuf` supports zig-zag encoded signed integers and unsigned integers using the same
compact varint scheme as the C library.

```java
import oyo.lol.util.DyBuf;

DyBuf buf = new DyBuf(32);
buf.putSignedVarLong(-123)
   .putUnsignedVarLong(300);

buf.flip();

System.out.println(buf.getSignedVarLong());   // -123
System.out.println(buf.getUnsignedVarLong()); // 300
```

Use `putSignedVarLong` / `getSignedVarLong` for negative values. Use
`putUnsignedVarLong` / `getUnsignedVarLong` for non-negative values. The same length
prefixing powers `putVarLengthData` / `getVarLengthData` and
`putVarLengthString` / `getVarLengthString`.

For compatibility with the C helpers that encode NUL-terminated c-strings, use
`putCStringWithVarLength` / `getCStringWithVarLength`. These append the trailing
`0x00` byte on write and strip it on read.

## Typdex markers

Typdex combines a logical type with an index in a compact marker. Java uses the same
canonical bit layout as C, Python, and JavaScript.

```java
import oyo.lol.util.DyBuf;

DyBuf buf = new DyBuf(32);
buf.putTypdex(new DyBuf.Typdex(DyBuf.TYPDEX_TYP_INT, 3));
buf.putTypdex(new DyBuf.Typdex(DyBuf.TYPDEX_TYP_STRING, 0x42));

buf.flip();

DyBuf.Typdex first = buf.getTypdex();
DyBuf.Typdex second = buf.getTypdex();

assert first.type == DyBuf.TYPDEX_TYP_INT;
assert first.index == 3;
assert second.type == DyBuf.TYPDEX_TYP_STRING;
assert second.index == 0x42;
```

`TYPDEX_TYP_OBJ` (`0x0e`) marks protocol-defined objects; its index and payload are
owned by the application protocol. `DYPE_F_*` mirrors the reserved dypkt function
indices.

## JSON helpers

`encodeJson` and `decodeJson` store JSON-compatible Java values using the
JSON-equivalent dybuf convention documented in
[`DYPKT_SCHEMA_CONVENTION.md`](../DYPKT_SCHEMA_CONVENTION.md). The Java value model is:

- `Map<String, Object>` for JSON objects. Use `LinkedHashMap` when encoded byte order
  matters.
- `List<Object>` for arrays.
- `String`, `Boolean`, `Long`, `Double`, and `null` for scalars.

```java
import oyo.lol.util.DyBuf;

import java.util.ArrayList;
import java.util.LinkedHashMap;
import java.util.List;
import java.util.Map;

Map<String, Object> first = new LinkedHashMap<>();
first.put("id", 1L);
first.put("name", "alpha");

Map<String, Object> second = new LinkedHashMap<>();
second.put("id", 2L);
second.put("name", "beta");

Map<String, Object> payload = new LinkedHashMap<>();
payload.put("items", new ArrayList<>(List.of(first, second)));

byte[] encoded = DyBuf.encodeJson(payload);
Object decoded = DyBuf.decodeJson(encoded);
```

Integers must fit JavaScript's safe integer range for cross-language parity.
Fractional numbers are encoded as doubles. `NaN` and infinities are rejected because
they are not valid JSON values.

## Version notes

- `0.5.0` is a wire-format compatibility release. Fixed-width floating point payloads
  now follow the same big-endian order as fixed-width integers. Data written by older
  little-endian C builds through the float/double helpers may need migration before
  reading with `0.5.0`.

## Developing locally

The current Java package intentionally has no Maven or Gradle wiring. Run the golden
fixture test from the repository root:

```bash
tools/generate_fixtures.sh
tools/test_java_fixtures.sh
```

`tools/test_java_fixtures.sh` compiles `DyBuf.java` and `FixtureTest.java` with
`javac`, then runs the fixture suite with `java`. The suite checks varints, typdex,
var-length bytes/strings, JSON golden cases, and deterministic random JSON round trips.

## Publishing

Java publishing is not wired yet. Before publishing, add a Gradle or Maven build that
produces the main jar, sources jar, javadoc jar, and runs the golden fixture suite.
Maven Central is the preferred long-term target; GitHub Packages or JitPack can be
used as transitional options.
